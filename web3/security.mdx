Web3 frontend cần bảo vệ **user funds** — connect wallet, sign transactions, manage approvals. Một bug = mất tiền user.

## Các mối đe dọa chính

| Mối đe dọa | Mô tả | Tác động |
|---|---|---|
| **Phishing** | Fake DApp, malicious signatures | Wallet drained |
| **Approval abuse** | Unlimited token approvals | Tokens stolen |
| **Malicious transactions** | Hidden contract calls | State manipulation |
| **Private key theft** | Local storage, browser extension | Total loss |
| **MEV attacks** | Front-running, sandwich | Trade losses |
| **Cross-site scripting** | XSS in DApp | Wallet theft |
| **DNS hijacking** | Wrong domain | Phishing |

## Wallet Connection Best Practices

```tsx
// ✅ CORRECT - Validate connection request
import { useAccount, useDisconnect } from 'wagmi';

function ConnectWallet() {
  const { address, isConnected } = useAccount();

  // Verify contract address before sending transactions
  const verifyContract = (addr: string): boolean => {

    return addr.toLowerCase() === EXPECTED.toLowerCase();
  };

  const handleTransaction = async () => {
    if (!isConnected) {
      alert('Please connect wallet first');
      return;

    }

    // Display clear transaction details before signing
    showTransactionPreview({

          <button onClick={disconnect}>Disconnect</button>
        </>
      ) : (
        <w3m-button />
      )}
    </div>
  );
}
```

## Token Approvals — Biggest Attack Surface

<Warning>
**80% DeFi hacks liên quan đến token approvals!** Default unlimited approval là rủi ro lớn nhất.
</Warning>

```solidity
// ❌ BAD - unlimited approval
IERC20(token).approve(spender, type(uint256).max);

// ✅ GOOD - exact amount
IERC20(token).approve(spender, amount);

// ✅ BETTER - use permit (EIP-2612)
function swapWithPermit(
    uint256 amount,
    uint256 deadline,
    uint8 v,
    bytes32 r,
    bytes32 s
) external {
    // No approve needed, single transaction
    IERC20Permit(token).permit(msg.sender, address(this), amount, deadline, v, r, s);
    // ... swap logic
}
```

### Manage Approvals với wagmi

```tsx
import { useAccount, useWriteContract } from 'wagmi';

function TokenApproval() {
  const { address } = useAccount();
  const { writeContract } = useWriteContract();

  // ❌ Avoid - unlimited approval
  const approveUnlimited = () => {
    writeContract({
      address: tokenAddress,
      abi: erc20Abi,
      functionName: 'approve',
      args: [spenderAddress, BigInt(2) ** BigInt(256) - BigInt(1)]
    });
  };

  // ✅ GOOD - exact amount only
  const approveExact = (amount: bigint) => {
    writeContract({
      address: tokenAddress,
      abi: erc20Abi,
      functionName: 'approve',
      args: [spenderAddress, amount]
    });
  };

  // ✅ BEST - revoke after use
  const revokeApproval = () => {
    writeContract({
      address: tokenAddress,
      abi: erc20Abi,
      functionName: 'approve',
      args: [spenderAddress, 0n]  // 0 = revoke
    });
  };
}
```

## XSS Prevention

```tsx
// ❌ DANGEROUS - rendering untrusted content
<div dangerouslySetInnerHTML={{ __html: userInput }} />

// ✅ SAFE - sanitize with DOMPurify
import DOMPurify from 'dompurify';

function SafeComment({ content }) {
  const clean = DOMPurify.sanitize(content);
  return <div dangerouslySetInnerHTML={{ __html: clean }} />;
}

// ✅ BETTER - use markdown library that handles XSS
import ReactMarkdown from 'react-markdown';

function MarkdownComment({ content }) {
  return (
    <ReactMarkdown
      skipHtml
      components={{
        // Custom component for links - force noopener
        a: ({ href, children }) => (
          <a href={href} target="_blank" rel="noopener noreferrer">
            {children}
          </a>
        )
      }}
    >
      {content}
    </ReactMarkdown>
  );
}
```

## Phishing Protection

```tsx
function useDomainVerification() {
  const EXPECTED = 'app.uniswap.org';
  const current = window.location.hostname;

  // ⚠️ Always verify you're on the real domain
  const isValid = current === EXPECTED ||
    current.endsWith('.uniswap.org');

  return isValid;
}

function App() {
  const valid = useDomainVerification();

  if (!valid) {
    return (
      <ErrorPage>
        ⚠️ PHISHING WARNING
        You're on "{window.location.hostname}" but expected "{EXPECTED}".
        Please verify the URL in your browser address bar.
      </ErrorPage>
    );
  }

  return <MainApp />;
}
```

## CSRF Protection for Signatures

```tsx
// ❌ VULNERABLE - signature can be replayed
async function signMessage(message: string) {
  const signature = await signer.signMessage(message);
  return signature;  // Attacker can replay on different action!
}

// ✅ SAFE - include nonce + domain + version
async function safeSignMessage(action: string) {
  const message = [
    'MyDApp Auth',
    'Version: 1',
    `Domain: ${window.location.host}`,
    `Action: ${action}`,
    `Nonce: ${nonce}`,        // Unique per request
    `Timestamp: ${Date.now()}` // Expiry check
  ].join('\n');

  const signature = await signer.signMessage(message);
  return { signature, message };
}
```

## Frontend Security Checklist

<Check>
- [ ] No private keys stored in browser
- [ ] All token approvals use exact amounts (not unlimited)
- [ ] Transaction details displayed before signing
- [ ] Contract addresses verified (whitelist known addresses)
- [ ] Domain verification on app load
- [ ] CSP headers set (no inline scripts)
- [ ] Subresource Integrity (SRI) for external scripts
- [ ] HTTPS enforced (HSTS headers)
- [ ] User-Agent check (block outdated browsers)
- [ ] CSP `frame-ancestors 'none'` (anti-clickjacking)
- [ ] Security headers: X-Content-Type-Options, X-Frame-Options
- [ ] MEV protection for high-value transactions
</Check>

## Smart Contract Integration Safety

```tsx
// Before any contract interaction, validate
async function safeContractCall(contractAddress, abi, functionName, args) {
  // 1. Check contract is verified on Etherscan
  const verified = await checkEtherscanVerification(contractAddress);
  if (!verified) {
    throw new Error('Contract not verified on Etherscan');
  }

  // 2. Check function signature matches expected
  const expectedSig = 'swapExactTokensForETH(uint256,uint256,address[],address,uint256)';
  if (getFunctionSelector(abi, functionName) !== expectedSig) {
    throw new Error('Function signature mismatch');
  }

  // 3. Use multicall for read operations (save RPC calls)
  const multicallResult = await multicall.call([
    { address: contractAddress, abi, functionName, args }
  ]);

  return multicallResult;
}
```
