## 3 khái niệm cốt lõi

<Steps>
  <Step title="Prompt — Mô tả yêu cầu">
    Mọi thứ bắt đầu từ một prompt rõ ràng. Claude Code càng nhận được nhiều thông tin (URL, fields, behavior), code tạo ra càng đúng ý.
  </Step>
  <Step title="Context — Bối cảnh dự án">
    Claude Code hiểu toàn bộ dự án: cấu trúc thư mục, file có sẵn, framework đang dùng. Cung cấp thêm cấu hình Firebase, design system, requirements càng tốt.
  </Step>
  <Step title="Iteration — Lặp lại cải tiến">
    Không phải lúc nào cũng đúng ngay từ lần đầu. Bạn mô tả phần sai (hiện tại vs mong muốn), Claude Code sửa lại.
  </Step>
</Steps>

## Quy trình làm việc điển hình

<Tip>
Quy trình mình khuyến nghị: **Setup → Thiết kế dữ liệu → Tính năng từng phần → Bảo mật → Deploy**
</Tip>

1. **Setup dự án** — Yêu cầu Claude Code khởi tạo với đầy đủ tech stack
2. **Thiết kế dữ liệu** — Mô tả cấu trúc cần lưu, để Claude Code đề xuất Firestore schema
3. **Xây từng tính năng** — Chia nhỏ thành các bước, kiểm tra sau mỗi bước
4. **Thêm bảo mật** — Yêu cầu viết Security Rules
5. **Deploy** — Kết nối Vercel và deploy
