## 1. Firestore — Cơ sở dữ liệu

### Firestore là gì?

Firestore tổ chức dữ liệu theo **Collections** (bộ sưu tập) chứa **Documents** (tài liệu).

<Tip>
**Phép so sánh:** Collection = Ngăn kéo · Document = Tờ hồ sơ trong ngăn kéo · Fields = Các mục trong hồ sơ
</Tip>

### Thiết kế cấu trúc dữ liệu

```
📁 Firestore Database
│
├── 👥 users (Collection)
│   ├── user_001 (Document)
│   │   ├── name: "Nguyễn Văn A"
│   │   ├── email: "a@gmail.com"
│   │   ├── role: "admin"
│   │   ├── department: "Engineering"
│   │   └── avatarUrl: "https://..."
│   └── user_002 (Document)
│
├── 📋 tasks (Collection)
│   ├── task_001 (Document)
│   │   ├── title: "Làm báo cáo Q1"
│   │   ├── description: "Tổng hợp số liệu..."
│   │   ├── status: "in_progress"
│   │   ├── priority: "high"
│   │   ├── assigneeId: "user_001"
│   │   ├── projectId: "project_001"
│   │   └── deadline: 2026-04-30
│   │
│   └── 💬 comments (Subcollection)
│       ├── comment_001
│       └── comment_002
│
└── 🏢 projects (Collection)
    └── project_001 (Document)
        ├── name: "Dự án ABC"
        └── members: ["user_001", "user_002"]
```

### Subcollections — Dữ liệu lồng nhau

| Subcollection | Collection riêng |
|---|---|
| Dữ liệu chỉ có ý nghĩa trong context document cha | Dữ liệu cần truy vấn độc lập |
| Comments của 1 task | Tasks (cần lọc, tìm tất cả) |
| Activity log của 1 task | Users (hiển thị ở nhiều nơi) |

## 5. Security Rules — Bảo mật dữ liệu

Security Rules là bộ quy tắc xác định ai được đọc/ghi dữ liệu nào. Chúng chạy trên server Firebase — không thể bypass từ client.

### Ví dụ Firestore Rules

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    match /users/{userId} {
      allow read: if request.auth != null;
      allow write: if request.auth.uid == userId;
    }

    match /tasks/{taskId} {
      allow read: if request.auth != null;
      allow create: if request.auth != null;
      allow update: if request.auth.uid == resource.data.assigneeId;
      allow delete: if request.auth.uid == resource.data.createdBy;
    }
  }
}
```

### Các cấp độ bảo mật

| Cấp | Quy tắc | Dùng khi nào |
|---|---|---|
| **Public** | Ai cũng đọc được | Trang công khai, landing page |
| **Authenticated** | Đã đăng nhập mới đọc/ghi | Mặc định cho hầu hết data |
| **Owner only** | Chỉ chủ sở hữu | Thông tin cá nhân, cài đặt riêng |
| **Role-based** | Theo vai trò (admin, member) | Quản lý users, xóa dữ liệu |
| **Team-based** | Theo nhóm/project | Tasks chỉ thấy bởi team members |
| **Deny all** | Chặn hoàn toàn từ client | Data chỉ Cloud Functions truy cập |

## Cách 5 thành phần phối hợp

Luồng tạo task hoàn chỉnh:

```
Frontend (tạo task)
  ↓
Services/auth.ts → Authentication xác thực
  ↓
Services/tasks.ts → Firestore ghi task document
  ↓
Cloud Functions trigger onTaskCreated
  ↓
Tạo notification + gửi email
  ↓
Security Rules kiểm tra quyền ở mọi bước
```

## Câu hỏi thường gặp

<AccordionGroup>
  <Accordion title="Có cần viết Security Rules ngay từ đầu không?">
    Khi tạo Firestore lần đầu, Firebase cho phép chọn test mode (mở 30 ngày) — đủ cho giai đoạn phát triển.

    **Trước khi deploy cho người dùng thật, BẮT BUỘC phải viết Security Rules.**
  </Accordion>
  <Accordion title="Firebase Emulator là gì?">
    Firebase Emulator là phiên bản Firebase chạy trên máy tính, mô phỏng Firestore, Auth, Storage, Functions mà không cần internet hay ảnh hưởng data thật.

    Dùng khi phát triển/test trên local, test Cloud Functions trước khi deploy.
  </Accordion>
  <Accordion title="Dữ liệu có mất khi Firebase gặp sự cố?">
    Firebase có SLA 99.95% uptime — thời gian ngừng hoạt động tối đa khoảng 4 giờ/năm.

    Dữ liệu được replicate trên nhiều data center. Firestore hỗ trợ backup tự động.
  </Accordion>
</AccordionGroup>
