Claude Code là công cụ lập trình AI agentic do Anthropic phát triển — không chỉ gợi ý code như các trợ lý thông thường, mà thực sự hành động: đọc file, chạy lệnh, sửa lỗi, chạy test và lặp lại cho đến khi hoàn thành nhiệm vụ. Thay vì bạn phải sao chép từng đoạn code từ chatbot vào editor, Claude Code hoạt động trực tiếp trong môi trường phát triển của bạn và xử lý cả một luồng công việc phức tạp theo yêu cầu.

## Vòng lặp agentic: Cách Claude Code suy nghĩ và hành động

Điểm cốt lõi khiến Claude Code khác biệt là kiến trúc **vòng lặp agentic** (agentic loop). Thay vì chỉ trả lời một lần rồi dừng, Claude liên tục lặp qua ba giai đoạn cho đến khi hoàn thành mục tiêu:

<Steps>
  <Step title="Thu thập ngữ cảnh (Gather Context)">
    Claude đọc codebase của bạn — file source, cấu hình, lịch sử git, file CLAUDE.md và bộ nhớ đã lưu — để hiểu toàn bộ bức tranh của dự án trước khi bắt tay vào bất kỳ hành động nào.
  </Step>
  <Step title="Thực hiện hành động (Take Action)">
    Claude dùng các công cụ tích hợp để thực thi: chỉnh sửa file, chạy lệnh shell, tra cứu tài liệu web, chạy test hay commit code lên git — tất cả diễn ra trong môi trường thực của bạn.
  </Step>
  <Step title="Kiểm tra kết quả (Verify Results)">
    Sau mỗi hành động, Claude xem xét kết quả: test có pass không? Có lỗi type mới nào không? Có cần điều chỉnh không? Nếu cần, Claude tiếp tục vòng lặp cho đến khi đạt được kết quả đúng.
  </Step>
</Steps>

<Note>
  Vòng lặp agentic có nghĩa là Claude có thể tự mình phát hiện và sửa lỗi mà không cần bạn can thiệp ở từng bước. Bạn giao mục tiêu — Claude tìm cách đạt được mục tiêu đó.
</Note>

## Các công cụ tích hợp sẵn

Claude Code đi kèm một bộ công cụ toàn diện để tương tác với dự án và môi trường của bạn:

| Danh mục | Khả năng |
| --- | --- |
| Thao tác file | Đọc file, chỉnh sửa code, tạo file mới, đổi tên và tổ chức lại |
| Tìm kiếm | Tìm file theo pattern, tìm nội dung với regex, khám phá codebase |
| Thực thi | Chạy lệnh shell, khởi động server, chạy test, dùng git |
| Web | Tìm kiếm web, lấy tài liệu, tra cứu thông báo lỗi |
| Code intelligence | Xem lỗi type và warning sau khi sửa, nhảy đến định nghĩa, tìm references |

## Claude Code có thể truy cập những gì trong dự án của bạn

Khi bạn khởi động Claude Code trong một thư mục dự án, Claude có thể làm việc với toàn bộ ngữ cảnh sau:

- **File source code** — tất cả các file trong dự án, không bị giới hạn bởi cửa sổ ngữ cảnh nhỏ như các editor plugin thông thường
- **Terminal và shell** — chạy bất kỳ lệnh nào bạn cho phép: `npm test`, `git log`, `docker ps`, v.v.
- **Trạng thái git** — xem diff, lịch sử commit, branch hiện tại và các thay đổi chưa được stage
- **File CLAUDE.md** — file hướng dẫn dự án mà bạn tạo ra để Claude hiểu quy ước, cấu trúc và yêu cầu đặc thù của dự án
- **Bộ nhớ tự động** — Claude tự lưu lại các phát hiện quan trọng (như cấu trúc thư mục, lệnh build, quy ước đặt tên) để dùng lại trong các phiên làm việc sau

<Tip>
  File `CLAUDE.md` đặt ở thư mục gốc dự án là cách hiệu quả nhất để "dạy" Claude về dự án của bạn — từ cách chạy test đến các quy ước code riêng của team.
</Tip>

## Claude Code khác gì so với các trợ lý code thông thường?

Nhiều developer đã quen với các công cụ như GitHub Copilot hay các plugin autocomplete — những công cụ này hoạt động ở cấp độ **dòng code** hoặc **hàm**. Claude Code hoạt động ở cấp độ **dự án và tác vụ**.

<Accordion title="Trợ lý inline (Copilot, autocomplete)">
  Các công cụ này nhìn thấy đoạn code xung quanh con trỏ của bạn và gợi ý dòng tiếp theo. Chúng rất hữu ích cho việc tự động điền, nhưng không thể thực thi lệnh, không đọc toàn bộ codebase và không tự sửa lỗi sau khi tạo code.
</Accordion>

<Accordion title="Claude Code (agentic)">
  Claude Code đọc toàn bộ dự án, hiểu kiến trúc tổng thể, thực thi lệnh trong terminal, kiểm tra kết quả và lặp lại. Bạn có thể giao cho Claude một nhiệm vụ cấp cao như *"Thêm tính năng xác thực JWT vào API của tôi"* và Claude sẽ tự tìm hiểu codebase, viết code, chạy test và báo cáo kết quả.
</Accordion>

## Models và lựa chọn sức mạnh xử lý

Claude Code sử dụng các model ngôn ngữ lớn của Anthropic. Bạn có thể chọn model phù hợp với từng tác vụ:

| Model | Đặc điểm | Phù hợp với |
| --- | --- | --- |
| **Claude Sonnet** (mặc định) | Cân bằng tốc độ và chất lượng | Công việc hàng ngày, refactor, viết test |
| **Claude Opus** | Suy luận mạnh hơn, chậm hơn | Bài toán phức tạp, kiến trúc hệ thống, debug khó |

Để chuyển model, dùng lệnh `/model` trong phiên làm việc hoặc flag `--model` khi khởi động:

```bash
# Chạy với Claude Opus
claude --model claude-opus-4-5

# Hoặc chuyển model trong phiên làm việc
/model claude-opus-4-5
```
