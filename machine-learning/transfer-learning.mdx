Transfer learning = dùng **pre-trained models** trên millions data → fine-tune trên data nhỏ hơn. Đây là paradigm chính của ML hiện đại.

## Why Transfer Learning?

<Check>
- ✅ Train nhanh hơn nhiều lần (hours thay vì weeks)
- ✅ Cần ít data hơn (100 samples thay vì 100K)
- ✅ Kết quả tốt hơn training từ đầu
- ✅ Đã được validate bởi cộng đồng
</Check>

## CLIP — Zero-shot Image Classification

```python
from transformers import CLIPProcessor, CLIPModel

model = CLIPModel.from_pretrained("openai/clip-vit-base-patch32")
processor = CLIPProcessor.from_pretrained("openai/clip-vit-base-patch32")

image = Image.open("cat.jpg")
text_labels = ["a photo of a cat", "a photo of a dog", "a photo of a car"]

inputs = processor(images=image, text=text_labels, return_tensors="pt")
outputs = model(**inputs)
logits = outputs.logits_per_image[0]

predicted = text_labels[logits.argmax()]
print(f"Predicted: {predicted}")
```

## LoRA — Parameter-Efficient Fine-tuning

```python
from peft import LoraConfig, get_peft_model

# LoRA — only train small number of parameters
lora_config = LoraConfig(
    r=16,           # rank
    lora_alpha=32,
    target_modules=["q_proj", "v_proj"],  # only attention layers
    lora_dropout=0.1
)

model = get_peft_model(model, lora_config)
print(f"Trainable params: {model.print_trainable_parameters()}")
# Shows: trainable: 0.1% of total params
```

## Comparison: When to Use What?

| Approach | Data needed | Training time | GPU needed |
|---|---|---|---|
| **From scratch** | 100K+ images | Days/weeks | High-end GPU |
| **Fine-tune last layers** | 1K-10K images | Hours | Mid-range GPU |
| **Fine-tune all layers** | 10K+ images | Hours-days | High-end GPU |
| **LoRA fine-tune** | 100-1K samples | Hours | Any GPU |
| **Zero-shot (CLIP/GPT)** | 0 samples | N/A | CPU or small GPU |

<Tip>
Bắt đầu với **zero-shot** hoặc **few-shot** (dùng pre-trained model trực tiếp). Nếu không đủ tốt → fine-tune. Nếu data quá ít → LoRA. Chỉ train từ đầu khi dùng domain-specific data rất lớn (medical, satellite, etc.).
</Tip>
