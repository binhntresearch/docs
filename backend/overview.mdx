Backend = **phần xử lý logic phía server** — authentication, database, business logic, API endpoints. Khác với frontend (UI), backend làm việc với data, APIs, và infrastructure.

## Backend là gì?

<Callout type="info">
Backend = tất cả những gì phía sau frontend giao tiếp. Frontend user clicks button → Backend xử lý → Database lưu trữ → Frontend hiển thị kết quả.
</Callout>

## Các thành phần chính

<CardGroup>
  <Card title="API Server" icon="server">
    Nhận HTTP requests, xử lý logic, trả response
  </Card>
  <Card title="Database" icon="database">
    Lưu trữ data — relational (Postgres) hoặc NoSQL (MongoDB)
  </Card>
  <Card title="Cache" icon="memory">
    Redis — tăng tốc bằng cách cache frequent queries
  </Card>
  <Card title="Message Queue" icon="envelope">
    RabbitMQ, Kafka — xử lý async tasks (email, notifications)
  </Card>
  <Card title="Storage" icon="box-archive">
    File storage — S3, GCS, Firebase Storage
  </Card>
  <Card title="Search" icon="magnifying-glass">
    Elasticsearch, Algolia — full-text search
  </Card>
</CardGroup>

## Backend Architecture Patterns

<Tabs>
  <Tab title="Monolith">
    Tất cả code trong 1 project. Dễ bắt đầu, khó scale.
    ```
    backend/
    ├── routes/
    ├── services/
    ├── models/
    └── app.ts
    ```
  </Tab>
  <Tab title="Modular Monolith">
    1 project nhưng tách thành modules rõ ràng. Tốt cho medium apps.
    ```
    backend/
    ├── users/
    ├── orders/
    ├── payments/
    └── shared/
    ```
  </Tab>
  <Tab title="Microservices">
    Nhiều services độc lập, giao tiếp qua APIs. Scale tốt nhưng phức tạp.
    ```
    User Service | Order Service | Payment Service
       ↓                ↓                ↓
       └────────────────┴────────────────┘
                  API Gateway
    ```
  </Tab>
  <Tab title="Serverless">
    Functions triggered bởi events. Pay-per-use.
    ```
    Vercel/Cloudflare Functions
    AWS Lambda
    Firebase Functions
    ```
  </Tab>
</Tabs>

## Best Practices

<Tip>
- **Project structure:** Tổ chức folders rõ ràng (routes, services, models, middleware)
- **Error handling:** Centralized error handler, custom error classes
- **Validation:** Validate input ngay tại boundary (DTO, Zod)
- **Logging:** Structured logs với correlation IDs
- **Testing:** Aim 80%+ coverage, especially cho business logic
- **Security:** Never trust input, sanitize output, use parameterized queries
- **Performance:** Cache aggressively, paginate everything, monitor metrics
</Tip>
